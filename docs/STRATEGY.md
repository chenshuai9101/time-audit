# 时间审计 · 商业化与落地策略

> 本文档固化一次深度讨论的结论，是后续所有取舍的基准。
> 定稿日期：2026-06-03

## 一句话定位（对外）

> **让 AI Agent 看懂"用户该自动化什么"的发现层。**
> 从真实屏幕行为里，带证据、带 ROI、带可执行性分层地告诉你/你的 Agent：哪些事在重复、哪些能交给 AI、哪些根本不该做。

## 选定的商业模式：开源底层为主线 + 细 B 输血

经过对四条路（卖个人产品 / 卖企业方案 / 卖服务 / 卖给 Agent 生态做底层）的权衡，选定：

**主线（路线 C）：开源底层 + 团队/企业版变现。**
- 开源 CLI/MCP/schema = 获客 + 信任 + 抢标准的渠道，**不是商业模式本身**。
- 钱来自顶上薄薄一层：有预算的**团队 / 企业版**（脱敏聚合、ROI 看板、托管、SSO、支持）。
- 个人版永远免费——使命是被用、被传、攒 star，不在个人版设收费点。

**输血（细路线 B）：每月 1–2 单付费"AI 落地诊断"。**
- 用自己的开源工具给中小企业做诊断、交付报告与自动化建议，按次/按项目收费。
- 作用有二：① 付房租，扛过开源 6–12 个月的现金缺口；② **反哺主线**——每单都在验证 schema、产出案例、喂 Fara 执行闭环数据。
- 这条 B 是"能活着的 C"的关键。**纯裸跑 C 而无输血 = 会在赚到钱前饿死**，明确不做。

## 四道壁垒（因为代码本身不是壁垒，1000 行可被一周复刻）

1. **抢标准** —— 让别人都按时间审计的"自动化机会 schema"来读"该自动化什么"（点/线/面 + evidence_sessions + automation_difficulty + 预计周节省）。这是最大的赌注（Stripe / Terraform / LangChain 打法）。
2. **抢品类名** —— "用 AI 审计自己的屏幕、发现该自动化什么" = 想到时间审计。
3. **隐私/本地优先品牌** —— 对 Rewind / Recall / 云端工具的真差异化，云模式的对手抄不动。
4. **数据网络效应（长期）** —— 脱敏聚合够多后，沉淀"医生/程序员/运营最常见可自动化流程 Top N"，别人买不到。

## 必须直视的风险

1. **没分发 = 死。** 单干需拿出约一半时间做内容/社区，而非写代码。讨厌传播 → C 会失败。
2. **现金缺口。** C 到第一笔像样收入 6–12 个月，与"要现金流"正面冲突——靠细 B 输血 + 个人 runway 扛过去。
3. **被大平台复刻。** 防御 = 标准 + 社区 + 速度 + 隐私品牌，且要快。
4. **14B 门槛锁天花板。** 已用双模型（云端可选）部分缓解。
5. **可能早 12 个月。** Agent/MCP 市场尚早，需边走边验证需求真实性。

## 终局

12 个月后：时间审计 = Agent 生态里"用户工作画像 / 自动化机会发现"的事实标准底层，开源驱动分发，团队/企业版变现，手握别人没有的真实工作流数据集。
